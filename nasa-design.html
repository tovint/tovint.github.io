NASA Time Travel

"Demonstrate how you can apply a common visual language for a NASA time travel app, which consists of a UI that identifies where in time you are, where in time you want to go, and an actionable affordance to initiate time travel (donâ€™t worry about flows). Provide a screen for both an iOS phone and an Android tablet that are on brand, while also being considerate of native platform design guidelines."

Initial Thoughts

When I looked at this design exercise, I had a lot of initial questions:

  • How is time travel being interpretted?: Is this time travel through space? Going to far off galaxies and planets (warping through space time). What about time travel here on earth? Should this app let people go back to different time periods in history? What about going forward in time?
  • Who will be using this app?: I was not sure if this app was an internal tool used for NASA employees, or if it was a genearal audience app accesible for the public. In addition, is the purpose of this app to be educational? If so, would the primary users be kids/ students?

Design Approach

In order to come up with a design for this app, I started by doing some research:

  • NASA Brand: I spent time looking at the NASA brand, and learning about the design guidelines. I used a document located on their website to learn more about their principles. I used the color tones from that document as the primary colors for the app. Looking through other NASA products I noticed that NASA uses a lot of pictures. They use imagery as a way to present information. I wanted to replicate this approach in my design as well. I wanted each different time to have a picture associated with it.
  • iOS VS Android: I went through the design guidelines for bot iOS and Android. Since both have different approaches, it was important to adhere to the design guidelines, but also maintain branding and a consistent style between both platforms. For android I chose to use Material Design components.

High Fidelity Designs

iPhone 6

This is the default screen that the user sees after loading the app. This screen highlights the current date at the top menu. On the left side of the menu is a hamburger icon which will pull out a side menu with more options. On the right side is an hour glass icon. This icon is used to represent time travel. Tapping on the icon will take you to the time travel screen, which always you to set your time parameters. The main content area shows popular times for the user to go to. You can mark a specific time as "Favorite" by tapping on the star icon associated with it. Tapping on a particular time will open that time with more detailed information.

This is the side menu that appears after tapping the hamburger icon. The side menu shows user information, as well as information on how many years the user has traveled in total and how many travels the user has done. Additionally the user can see all the times they have marked as "Favorite Times", and see previous destinations by tapping on "Past Visits".

If the user taps on the hour glass icon the time travel screen slides up. This is where the user can select where they want to travel to. This screen indicates the current time the user is at. In addition, once the user selects a time, he/she is given information on the travel distance, whether or not they have been to this time, and if they are going forward or backwards in time. After selecting where to go, clicking on "Start" will initiate time travel. As a fun addition, the user can instead click on "I'm feeling lucky" and be taken to a random moment in time (Inspiration from Google).

After clicking on "Start", the user is taken to the time they have selected. The time appears in the top menu bar. If there is an available moment in time it will appear as the first result. Other "Related Times" will also appear below to give the user more options. These times will be related based on the decade. Since the user selected to go to the 1989, the "Related Times" sections will show other moments from the 1980's.

Android Tablet (9")

This is the default screen for the Android Tablet. Used similar color schemes and layout as the iPhone 6 design. The current date is indicated in the top menu bar. For the android tablet I went with Material Design principles. The hour glass icon is inside a "Floating Action Button". Since this is the main action I wanted it to standout on a larger screen. Since, the tablet offers more room to work with I decided to add pre-text to each card in addition to just the title and date.

The side menu comes out when clicking on the hamburger icon. Layout is consistent with the iPhone 6 design.

After tapping on the "FAB" button you are presented with the time travel screen.

After clicking on the "Start" button the user is show the time they selected. Since the top bar menu has more space than the iPhone 6 I decided to add the "Welcome to" text before the date. I thought this would be a warm message for the user to see as well as depicate the notion of having taveled somewhere.
